## 1. Executive Summary

AI 职业智囊是一个基于多Agent系统的Web端职业决策产品，面向"还没辞职但在想"的B2B企业软件从业者。用户上传简历后，通过职业认知、岗位匹配、面试辅导三个递进模块，获得从自我认知到实战准备的完整职业决策支持。当评估结论为"不该跳槽"时，进入当前工作辅导模块。

**核心差异化：**
- 多模型并发圆桌讨论（非单线程模拟，每个角色用不同AI模型）
- AI效能专家角色（所有模块参与，评估AI对职业的影响）
- 动态画像模板（按岗位/行业/职级定制，非固定结构）
- Agent 动态占比架构（每个 Agent 有固定动态占比，模板+岗位知识+定制认知三层组装）
- 搜索增强（触发式搜索岗位/公司最新数据，搜索结果分级存储）

**核心价值主张：** "你在市场上值多少？下一步怎么走？AI怎么影响你的职业？"

## 2. Problem Statement

### 谁有这个问题？

3年以上工作经验、年薪15-25万、在B2B企业软件/互联网中厂/SaaS行业工作、尚未离职但对当前职业状态有疑虑的职场人。

### 问题是什么？

1. **信息不对称**：不知道自己在市场上值多少钱、处于什么位置
2. **决策焦虑**：想走但不敢走，怕走错、怕降薪、怕错过机会
3. **行动盲区**：即使决定跳，不知道简历怎么改、面试怎么准备；即使决定留，不知道怎么在当前岗位突破

### 为什么痛？

- 职业辅导老师：水平参差不齐，同一方法论面对所有人，无法跨行业
- ChatGPT/豆包：缺少用户画像、缺少完整业务流、信息易丢失
- 猎聘/简历网：站在甲方视角，没有职业辅导，没有JD打分

### 证据

- 目标用户（帆软内部员工）处于职业生涯十字路口，有真实需求
- 小红书上"职业规划"、"跳槽"内容极度活跃，说明需求存在
- 市面上职业辅导老师客单价3000-10000元，但交付质量参差不齐

### 排除用户

- 应届生（数据不足）
- 年薪50万+高管（超出AI能力）
- 纯躺平不焦虑的人（无付费动力）
- 能熟练使用Dify/Coze的20%用户（自己能搭）
- 追求"保offer"的用户（不是本产品范围）

### Jobs-to-be-done

1. "告诉我现在在市场上值多少"
2. "帮我判断该不该跳"
3. "如果要跳，帮我匹配岗位、准备面试"
4. "如果不该跳，帮我规划当前工作怎么突破"

## 4. Strategic Context

### 业务目标（副业）

| 指标 | 6个月目标 |
|------|---------|
| MAU | 100 |
| 月收入 | ≤5万 |
| 报告满意度 | ≥4.0/5 |
| 付费转化率 | ≥15% |
| 转介绍率 | ≥25% |

### 市场机会

- **目标人群**：3年+、15-25万年薪、B2B企业软件从业者
- **市场空白**：80%用户不会用高级AI工具，嫌豆包太泛，买不起高端猎头
- **竞争壁垒**：画像即壁垒（用户投入越多，切换成本越高）+ 站在用户侧而非雇主侧

### 竞品对比

| 竞品 | 问题 | 我们的差异 |
|------|------|-----------|
| 职业辅导老师 | 水平参差，无法跨行业 | AI多Agent系统，跨行业个性化 |
| ChatGPT/豆包 | 无画像、无业务流、无子Agent | 持久化画像 + 结构化流程 + 多Agent协作 |
| 猎聘/简历网 | 甲方视角，无辅导 | 用户视角 + JD打分 + 面试辅导 |

### 为什么是现在？

- AI Agent技术成熟（GPT-4o + 多Agent编排）
- 目标用户对AI工具的接受度提升
- 创始人在B2B企业软件行业有判断力（产品运营、销售、售前、市场、PMM、项目经理）

## 5. Solution Overview

### 产品架构

```
首页（三个入口 + 用户旅程图）
├── 入口A：职业认知（模块一）
├── 入口B：岗位匹配（模块二）
└── 入口C：面试辅导（模块三）

模块一完成后 → 圆桌结论判断
├── "该跳" → 模块二 → 模块三
└── "不该跳" → 模块零（当前工作辅导）

所有模块共享用户画像系统
```

### 四个模块

| 模块 | 目标 | 用户耗时 | 子Agent | 圆桌 |
|------|------|---------|---------|------|
| 模块零：当前工作辅导 | 20分钟内获得提升方案 | 20分钟 | 4个（职业导师+猎头+头部企业专家+AI效能专家） | 1次（辩论式） |
| 模块一：职业认知 | 15分钟内从"模糊焦虑"到"清晰定位" | 15分钟 | 4个（对话引导+画像构建+市场对标+AI效能专家） | 2次（辩论式+投票式） |
| 模块二：岗位匹配 | 5分钟内知道岗位适配度 | 5分钟 | 3个（岗位洞察+行业总监+猎头） | 1次（辩论式） |
| 模块三：面试辅导 | 20分钟内完成模拟面试 | 20分钟 | 2个（面试辅导+AI效能专家） | 2次（出题+评估） |

### Agent系统：12个Agent（动态占比模型）

**Agent 合并记录：**
- 面试官 + 答案优化 → **面试辅导 Agent**（参与圆桌，贡献面试官视角）
- 行业专家 + 岗位洞察 → **岗位洞察 Agent**（同时具备行业+岗位洞察能力）

**动态占比模型：** 每个 Agent 有固定的动态占比，决定其输出中有多少是运行时生成的。

| 动态占比 | Agent | 说明 |
|---------|-------|------|
| 0% | 圆桌主持、心理学家 | 纯静态，所有用户共用 |
| 10-30% | 职业导师(10%)、画像构建(20%)、对话引导(30%) | 框架+少量定制 |
| 50-60% | 猎头(50%)、市场对标(50%)、AI效能专家(60%) | 框架+岗位知识 |
| 100% | 行业总监、头部企业专家、岗位洞察、面试辅导 | 完全按用户生成 |

**Agent 架构：** Agent = 模板（静态框架）+ 岗位知识（可复用）+ 定制认知（一客一用）
- 岗位知识库：positions/ 目录，按岗位存储 JSON，所有用户复用
- 定制认知：运行时根据用户简历+目标公司实时生成
- 搜索增强：触发式搜索（新岗位/目标公司/数据过期），搜索结果分级存储
- 100% 动态 Agent 生成流程：见 kitty-specs/v1-career-cognition/agents/DYNAMIC-ROLE-GENERATION-PROCESS.md
- 编排式架构（Orchestrator），非自主通信
- 圆桌讨论采用多模型并发架构（AgentRound），每个角色用不同AI模型

**V1 支持岗位：** 数据分析师、B2B销售

### 圆桌讨论机制

**圆桌讨论次数：**
- **全部模块：5次（4+1）**
  - 模块一：2次（画像模板生成圆桌4角色 + 职业方向圆桌6+1）
  - 模块二：1次岗位匹配圆桌（3角色：岗位洞察+行业总监+猎头）
  - 模块三：2次（出题圆桌 + 评估圆桌，2角色：面试辅导+AI效能专家）
  - 模块零：1次工作辅导圆桌（4角色：职业导师+猎头+头部企业专家+AI效能专家）

**圆桌类型：**

| 类型 | 架构 | 适用场景 | 说明 |
|------|------|---------|------|
| 辩论式 | Vercel AI SDK + Promise.allSettled（自研） | 模块一/二/三分析讨论 | 3个模型同时生成，交叉注入上下文 |
| 投票式 | Vercel AI SDK + Promise.allSettled（自研） | 模块一结论判断 | 独立分析→同行评议→主席综合 |

**强制分歧机制（2026-06-15 新增）：**
- 圆桌讨论中，每个模型必须提出至少一个不同观点或质疑，防止观点趋同
- 心理学家 Agent 承担"质疑者"角色，专门提出负面视角和尖锐质疑
- 综合阶段，主持 Agent 需要明确列出"分歧点"和"未被质疑的共识"

**搜索增强使用场景：**
- 薪资定位：搜索目标岗位实时薪资数据，补充模型推断
- 岗位匹配：搜索目标公司最新JD和团队信息
- 市场对标：搜索行业报告和竞品数据

### 用户画像系统：动态模板架构

- 三层结构：基础通用（固定）+ 岗位动态（按岗位生成）+ AI能力（固定+动态）
- 画像模板在简历解析后由圆桌会议动态生成
- 每个岗位/行业/职级的画像结构不同
- 取值范围、权重、计算公式由模板定义
- 更新机制：对话结束自动更新 + 10-15轮提醒 + 5天超期清理

**知识体系（2026-06-12 新增）：**
- 岗位知识库：positions/ 目录，按岗位存储 JSON，所有用户复用
- 定制认知：运行时根据用户简历+目标公司实时生成，一客一用
- 冷启动策略：LLM 通用知识 → 用户交互沉淀 → 岗位知识逐步完善
- 三层知识架构：global_knowledge.json（跨行业通用）+ industry_context.json（B2B行业通用）+ positions/{position}.json（岗位特定）

**多段经历支持（2026-06-12 新增）：**
- 用户可能有 2-3 段不同岗位/行业的经历（如：数据分析师→数据产品经理→B2B销售）
- 画像新增 `career_segments[]` 数组：每段经历独立记录（岗位、行业、公司、年限、核心技能、成就、离职原因）
- 画像新增 `career_narrative`：贯穿多段经历的主线叙事（由对话引导 Slot 7 收集）
- 画像新增 `composite_profile`：多段经历的组合稀缺度评估（稀有组合描述、稀缺度、市场价值倍数）
- 知识卡加载：多段经历用户加载多张 positions/{position}.json（最多 3 张），合成 composite_profile
- 设计原则：多段经历是"稀缺的复合型人才"，不是"经历不稳定"
- 理论支撑：Savickas 职业建构理论（叙事建构）+ Cal Newport 职业资本（稀缺组合）+ Scott Adams 人才栈（技能组合）

### 对话状态管理

- 三层记忆架构：用户画像（持久化）+ 对话上下文（最近5-8轮）+ 原始记录（归档）
- 输出依赖用户画像，而非压缩上下文
- 对话session 5天超期后，调用neat-freak提取关键信息到画像，然后删除session
- V1只支持电脑端，不支持多设备同步

| 风险 | 概率 | 影响 | 缓解 |

| 用户网络断开 | 低 | 低 | 自动重连，继续对话 |

## 11. V1 开发方案（2026-06-09确认）

### 11.1 已确认决策

| # | 决策 | 结果 | 说明 |
|---|------|------|------|
| 1 | Agent总数 | 12个（动态占比模型） | 面试官+答案优化→面试辅导；行业专家+岗位洞察→岗位洞察；每个Agent有固定动态占比(0%-100%) |
| 2 | V1圆桌讨论次数 | 6次 | 模块一2次（画像模板+职业方向）+ 模块二1次 + 模块三2次（出题+评估）+ 模块零1次 |
| 3 | 简历格式 | 仅PDF | V1不支持Word/图片 |
| 4 | 圆桌主持 | 包含 | 12个Agent中包含圆桌主持 |
| 5 | 财务顾问Agent | 不需要 | 对圆桌无核心价值 |
| 6 | 圆桌讨论架构 | 多模型并发 | AgentRound架构，成本接受 |
| 7 | 用户认证 | 邮箱注册 | 不支持匿名，每个用户只能看自己的内容 |
| 8 | V1支持岗位 | 数据分析师、B2B销售 | PMM及其他岗位V2扩展 |
| 9 | Landing Page | 简单产品介绍 | 不含用户旅程图 |
| 10 | 画像模板生成 | 职业导师生成通用模板 + 圆桌讨论生成定制化模板 | 3-5分钟，优先5分钟 |
| 11 | 竞争力评估 | 定性评估（强/中/弱）+ 内部严谨流程 | 每个Agent独立评估，BARS锚定，5阶段流水线，3次运行取共识 |
| 12 | 薪资分位 | 公开数据优先 | 最新月份数据，用户可补充。薪资是有数据锚点的量化指标 |
| 13 | 评估维度 | 每个Agent有独立BARS rubric | 维度评分1-5分，加权聚合后映射到强/中/弱 |
| 14 | 评估结果 | 定性评级+一句话定位+各Agent评语 | 不给0-100分，不给伪百分位。给可操作的洞察 |
| 15 | neat-freak机制 | 自动提炼，用户可查看 | 每15轮提醒一次 |
| 16 | 前端状态管理 | 流式输出 + 打字机效果 | 显示"正在输入..."，100ms/字 |
| 17 | 数据库迁移 | 每次迁移前自动备份 | 保留最近7天备份 |
| 18 | Agent架构 | 动态占比模型 | 模板+岗位知识+定制认知三层组装 |
| 19 | 岗位知识初始化 | 冷启动LLM+用户沉淀 | 用dynamic.md流程生成，不预置行业卡片 |
| 20 | 搜索增强 | 触发式搜索 | 新岗位/目标公司/数据过期时触发，搜索结果分级存储 |
| 21 | 定制认知 | 一客一用 | 根据用户简历+目标公司实时生成，不复用 |
| 22 | neat-freak冲突 | 标记待确认（方案B） | 5天超期清理时，若neat-freak提炼的信息与用户画像冲突，存入pending_updates表，下次对话时弹窗让用户确认（保留旧/用新的/合并）。参考mem0和ChatGPT记忆功能。工作量1-2天 |
