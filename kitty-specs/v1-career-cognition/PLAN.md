# V1 开发计划

> 版本：v3.0
> 日期：2026-06-17
> 基于：SPEC.md v3.0（全功能：模块一+模块零+模块二+模块三）
> 总耗时：44天（6周，副业节奏约7-8周）

---

## 1. 整体架构

V1 采用**五层开发方案**，先建基础架构，再改造 Agent 架构，然后做业务逻辑，再优化体验，最后实现模块二+三。

```
阶段0：项目骨架（3天）
  ↓
阶段0.5：架构改造（3天）— Agent模板拆分、岗位知识库、搜索策略
  ↓
第一层：基础架构（7天）— 用户能注册、能对话、能暂停恢复
  ↓
第二层：业务逻辑（10天）— 画像模板、竞争力评估（定性）、薪资定位（量化）
  ↓
第三层：体验优化（8天）— 自动提炼、流式输出、数据库迁移
  ↓
第四层：模块二+三（10天）— JD匹配、面试辅导
  ↓
阶段9：测试+上线（3天）— 功能测试、体验优化、部署上线
```

**核心原则：**
- 每层完成后都有可交付的功能
- 圆桌讨论 token 无上限（核心体验不打折）
- 其他模块分层模型优化（控制成本）
- Agent 动态占比模型：模板保下限，动态拔上限
- 岗位知识冷启动：LLM 通用知识 → 用户沉淀迭代

## 3. 第一层：基础架构（7天）

**目标：** 用户能注册登录、能进行对话、能暂停和恢复。

### 3.1 用户认证系统（3天）

| 任务 | 说明 | 耗时 |
|------|------|------|
| A1 注册功能 | 邮箱+密码注册，6位验证码验证邮箱 | 1天 |
| A2 登录功能 | 邮箱+密码登录，JWT token，单设备踢人 | 1天 |
| A3 密码重置 | 邮件验证码+新密码 | 0.5天 |
| A4 前端页面 | 注册、登录、验证、忘记密码页面 | 0.5天 |

**数据库表：** users、email_verifications、device_sessions
**依赖：** 阶段 0
**输出：** 用户能注册、登录、重置密码

### 3.2 对话状态流转图（2天）

| 任务 | 说明 | 耗时 |
|------|------|------|
| B1 数据库设计 | dialogue_sessions 表（含 slot_state、recent_window、initial_findings 字段）+ dialogue_messages 表 | 0.5天 |
| B2 状态管理 API | start/pause/resume/status 接口 + 增量提取逻辑 | 1天 |
| B3 前端集成 | 关闭页面自动暂停，回来提示恢复 | 0.5天 |

**状态定义：** active → paused → expired → archived
**上下文保留策略：** 结构化状态 + 增量提取（Slot 状态注入 prompt，原文滑动窗口 3 轮）
**依赖：** 阶段 0
**输出：** 对话可暂停、可恢复，Slot 状态无损保留

### 3.3 暂停/恢复/超期清理（2天）

| 任务 | 说明 | 耗时 |
|------|------|------|
| C1 定时任务 | 每天凌晨2点检查超期session | 1天 |
| C2 通知机制 | 站内+邮件通知，48小时后自动清理 | 1天 |

**超期策略：** 5天超期 → 通知用户 → 48小时等待 → 提取关键信息到画像 → 删除session
**依赖：** B2
**输出：** 对话超期自动清理，关键信息不丢失

**第一层里程碑：** M1 — 用户能注册登录，对话可暂停恢复

## 4. 第二层：业务逻辑（10天）

**目标：** 画像模板动态生成、竞争力评估（定性）、薪资定位（量化锚点）。

**多段经历支持（2026-06-12 新增）：**
- 画像新增 career_segments[]（多段经历数组）、career_narrative（职业叙事）、composite_profile（组合画像）
- 对话引导新增 Slot 7（职业叙事维度）
- 知识卡加载：多段用户加载多张 positions/*.json（最多 3 张）
- 各 Agent 新增多段分析逻辑（详见各 Agent prompt 文件）
- 设计原则：多段经历 = 稀缺复合型人才，不是经历不稳定
- 理论支撑：Savickas 职业建构 + Cal Newport 职业资本 + Scott Adams 人才栈

### 4.1 画像模板生成（3天）

| 任务 | 说明 | 耗时 |
|------|------|------|
| D1 通用基础模板 | 职业导师Agent生成基础层（即时） | 1天 |
| D2 定制化模板 | 圆桌讨论生成岗位动态层（3-5分钟） | 1.5天 |
| D3 模板合并 | 通用+定制→最终模板，存入数据库 | 0.5天 |

**三层结构：**
- Layer 1 基础通用（固定）：身份、动机、价值排序、风险偏好
- Layer 2 岗位动态（按岗位生成）：核心能力、行业特定、市场供需
- Layer 3 AI能力（固定+动态）：AI素养、替代风险、增效机会

**参与角色：** 动态岗位Agent + 职业导师Agent + 心理学家Agent + AI效能专家Agent
**依赖：** 阶段 0（AI SDK）
**输出：** 每个用户有定制化的画像模板

### 4.2 竞争力评估算法（3天）（2026-06-12 重构）

| 任务 | 说明 | 耗时 |
|------|------|------|
| E1 多Agent评估 | 5个Agent独立评估，BARS rubric + CoT推理 + 3次运行取共识 | 2天 |
| E2 圆桌讨论综合 | 各Agent亮明评级→交叉质疑→达成共识 | 0.5天 |
| E3 前端展示 | 定性评级+一句话定位+各Agent评语+薪资分位 | 0.5天 |

**评估机制（定性评估 + 内部严谨流程）：**
- 输出：5/5优秀、4/5良好、3/5中等、2/5待提升、1/5需重点提升 + 一句话定位 + 各Agent评语
- 不输出：0-100分、伪百分位
- 内部流程：证据收集 → BARS维度评分(1-5) → 加权聚合 → 定性映射 → 3次运行取共识
- 每个Agent有独立的BARS rubric（行为锚定描述）
- 仅薪资分位使用量化数字（有数据锚点）

**依赖：** D3（画像模板）
**输出：** 强/中/弱评级 + 一句话定位 + 各Agent评语 + 薪资分位 + 推理链

### 4.3 薪资分位数据源（2天）

| 任务 | 说明 | 耗时 |
|------|------|------|
| F1 数据收集 | 公开数据优先（猎聘、BOSS直聘、行业报告） | 1天 |
| F2 分位计算 | P25/P50/P75/P90 + 薪资区间 + 分布 | 1天 |

**数据来源优先级：** 公开数据 → 用户补充 → AI推断
**依赖：** 阶段 0（数据库）
**输出：** 薪资分位定位 + 市场区间

### 4.4 评分公式和权重（2天）

| 任务 | 说明 | 耗时 |
|------|------|------|
| G1 权重生成 | 多Agent讨论权重分配，圆桌综合 | 1天 |
| G2 前端展示 | 各维度权重+权重分配理由，用户可见 | 1天 |

**权重特点：** 不同岗位权重不同，一客一议，用户可见权重和理由
**依赖：** E1（多Agent评分）
**输出：** 透明的权重分配 + 理由说明

**第二层里程碑：** M2 — 用户上传简历后能看到定制化画像模板、竞争力评估（强/中/弱）、薪资定位
