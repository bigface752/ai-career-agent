# Agent 生成标准流程

> 版本：v1.0 | 日期：2026-06-09
> 用途：标准化 Agent Prompt 的生成过程，确保每个 Agent 都经过深度思考
> 调用方式：告诉 Claude "按 AGENT-GENERATION-PROCESS.md 的流程，生成 [Agent名称]"

---

## 流程总览

```
Phase 1: 定义 → 这个 Agent 是谁？
Phase 2: 搜索 → 行业里怎么做？
Phase 3: 圆桌 → 多角色激烈辩论
Phase 4: 提炼 → 核心认知和设计原则
Phase 5: 生成 → 写最终 Prompt
Phase 6: 校验 → 与其他 Agent 的边界检查
```

---

## Phase 1：定义（5分钟）

### 1.1 回答 5 个基本问题

| # | 问题 | 说明 |
|---|------|------|
| 1 | **这个 Agent 是谁？** | 角色名、一句话定位 |
| 2 | **他为谁服务？** | 服务哪个模块、服务什么用户场景 |
| 3 | **他解决什么问题？** | 用户的核心痛点 |
| 4 | **他和谁配合？** | 与其他 Agent 的协作关系 |
| 5 | **他不做什么？** | 明确边界，避免职责重叠 |

### 1.2 输出：Agent 定义卡

```markdown
| 维度 | 内容 |
|------|------|
| 角色名 | |
| 一句话定位 | |
| 所属模块 | |
| 核心任务 | |
| 与谁配合 | |
| 不做什么 | |
```

## Phase 2：搜索（10-15分钟）

### 2.1 GitHub 搜索

**⚠️ 时效性要求：必须搜索最近一个季度的最新信息，不使用过期过时的内容。优先筛选 2026 年更新的项目和数据。**

**搜索关键词模板：**
```
"[角色相关英文名]" + "framework"
"[角色相关英文名]" + "AI agent"
"[角色相关英文名]" + "tool"
"[角色相关英文名]" + "system"
```

**示例（职业导师）：**
```
"career coaching AI"
"career mentor agent"
"career development framework"
"career path planning"
"skills gap analysis career"
```

### 2.2 专业知识搜索

**搜索领域：**
- 学术理论（如：职业发展理论、心理学框架）
- 行业实践（如：大厂的职业阶梯框架）
- 专业方法论（如：GROW 教练模型、SWOT 分析）
- 数据来源（如：O*NET、ESCO 职业分类）

### 2.3 输出：搜索结果摘要

```markdown
## 搜索结果摘要

### GitHub 发现
| 项目 | 核心方法 | 对我们的启发 |
|------|---------|------------|

### 专业理论
| 理论/框架 | 核心内容 | 如何应用 |
|----------|---------|---------|

### 关键发现
1.
2.
3.
```

## Phase 3：圆桌会议（核心环节，20-30分钟）

### 3.1 参与角色（固定 5 个）

| 角色 | 视角 | 关注点 |
|------|------|--------|
| **产品经理** | 用户体验和产品价值 | 用户需要什么？怎么交付？ |
| **领域专家** | 专业知识和方法论 | 行业标准是什么？最佳实践？ |
| **目标用户代表** | 用户心理和实际场景 | 用户怎么想？什么会让他们信任？ |
| **AI工程师** | 技术实现和约束 | 能做吗？60秒够吗？成本？ |
| **行业从业者** | 行业特定知识 | 这个行业有什么特殊情况？ |

### 3.2 讨论结构（4 轮）

**Round 1：核心价值**
- 这个 Agent 的核心价值是什么？
- 用户为什么要信任他？
- 他和市面上的工具（ChatGPT等）有什么区别？

**Round 2：能力边界**
- 他必须具备什么能力？
- 他绝对不能做什么？
- 他的能力上限在哪？

**Round 3：输出设计**
- 输出什么内容？
- 输出格式怎么设计？
- 输出顺序怎么排？（先结论后分析？先优势后短板？）

**Round 4：协作和差异化**
- 和其他 Agent 怎么配合？
- 和其他 Agent 的边界在哪？
- 我们的差异化是什么？

### 3.3 输出：圆桌讨论记录

```markdown
## 圆桌讨论记录

### Round 1：核心价值
**[角色1]：** ...
**[角色2]：** ...
**共识：**
**分歧：**

### Round 2：能力边界
...

### Round 3：输出设计
...

### Round 4：协作和差异化
...
```

## Phase 5：生成（15-20分钟）

### 5.1 文件结构模板

```markdown
# [Agent名称]

> 角色类型：固定/动态
> 所属模块：[模块名]
> 状态：待开发
> 版本：v1.0 | 日期：YYYY-MM-DD

---

## 1. 角色定位
- 一句话定位
- 不是什么
- 是什么

## 2. 核心理念
- 设计原则（3-5条）
- 每条原则有对比表说明

## 3. 理论基础
- 使用的理论/框架
- 每个理论如何应用

## 4. 工作流程
- 输入 → 步骤 → 输出
- 用流程图或步骤说明

## 5. 分析维度
- 维度列表
- 每个维度的评估方法和数据来源

## 6. 输出结构
- JSON 格式
- 可截图卡片格式（如有）

## 7. System Prompt
- 完整的 System Prompt

## 8. 与其他角色的协作
- 协作关系表

## 9. 设计决策记录
- 关键决策及原因
```

### 5.2 System Prompt 编写规范

```
## System Prompt 结构

1. 角色定义（1-2句话）
2. 核心理念（3-5条，每条一句话）
3. 输入说明
4. 分析维度/工作流程
5. 输出格式（JSON）
6. 规则/红线
7. 与其他角色的协作
```

### 5.3 输出格式规范

```json
{
  // 必须字段
  "summary": "一句话结论",

  // 分析字段
  "dimensions": [
    {
      "name": "维度名",
      "score": 0-100,
      "evidence": "证据",
      "detail": "一句话分析"
    }
  ],

  // 建议字段
  "recommendations": [
    {
      "priority": 1,
      "action": "具体行动",
      "timeline": "时间线",
      "expected_outcome": "预期效果"
    }
  ],

  // 元数据
  "metadata": {
    "data_source": "数据来源",
    "confidence": "置信度",
    "note": "备注"
  }
}
```

## Phase 6：校验（5分钟）

### 6.1 边界检查清单

| 检查项 | 通过？ |
|--------|--------|
| 与 AI效能专家 的边界是否清晰？ | ⬜ |
| 与 心理学家 的边界是否清晰？ | ⬜ |
| 与 职业导师 的边界是否清晰？ | ⬜ |
| 与 市场对标 Agent 的边界是否清晰？ | ⬜ |
| 与 圆桌主持 的边界是否清晰？ | ⬜ |
| 输出格式是否与其他 Agent 一致？ | ⬜ |
| System Prompt 是否包含红线？ | ⬜ |

### 6.2 质量检查清单

| 检查项 | 通过？ |
|--------|--------|
| 每个评分是否有证据支撑？ | ⬜ |
| 每个建议是否有时间线？ | ⬜ |
| 是否避免了"建议提升能力"这种废话？ | ⬜ |
| 输出是否可截图分享？ | ⬜ |
| 是否标注了数据来源和置信度？ | ⬜ |
| 是否考虑了 B2B 行业特殊性？ | ⬜ |
