# 模块零：当前工作辅导

> **文件职责**：模块零专项。模块零的角色、流程、报告模板都记录在这里。
> 当职业认知结论是"不该跳"时，进入本模块

---

## 触发条件

```
模块一（职业认知）完成后
    ↓
圆桌讨论给出结论
    ↓
┌─────────────────────────────────────────────┐
│  结论判断：                                   │
│                                              │
│  · "建议跳槽" → 进入模块二（岗位匹配）         │
│  · "建议观望" → 进入模块零（当前工作辅导）      │
│  · "不该跳"   → 进入模块零（当前工作辅导）      │
└─────────────────────────────────────────────┘
```

**关键设计：结论必须明确。** 不能给用户一个模糊的"看情况"。圆桌讨论必须输出一个清晰的判断：跳 / 观望 / 不跳。这个判断要慎重、谨慎，但必须给。

---

## 目标

> **用户在 20 分钟内获得一份清晰的"当前工作提升方案"：我在当前岗位的定位是什么、我的天花板在哪、我要怎么往上走、具体做什么。**

## 子Agent设计（4个角色）

> 注：当前岗位管理者已由职业导师覆盖（天花板分析+卡点诊断已包含"管理者视角"的分析）

### Agent A：职业导师 Agent（固定）

**做什么：** 从职业发展全局视角，分析用户的天花板和突破路径。

**System Prompt 核心：**
```
你是一位资深职业导师，帮助过1000+职场人做职业发展。

你的任务不是帮用户跳槽，而是帮用户在当前岗位上做得更好。

分析维度：
1. 天花板分析：这个岗位的上限是什么？（薪资、职级、影响力）
2. 突破路径：从当前位置到天花板，需要经历哪些阶段？
3. 关键卡点：每个阶段最难突破的是什么？
4. 时间窗口：哪些提升有时间敏感性？（如35岁前晋升管理层）
```

**核心输出：**
- 当前岗位的职业发展路径图
- 用户在路径上的位置
- 下一步的关键突破点
- 时间窗口提醒

---

### Agent C：猎头 Agent（动态）

**做什么：** 从外部市场视角，分析用户当前岗位的市场价值和竞争力。

**动态生成逻辑：**
```
输入：用户的岗位 + 行业
生成：一个专注于该领域的资深猎头

示例（用户是B2B销售）：
"你是一位专注B2B企业软件领域的资深猎头，从业12年，
成功推荐过200+销售总监/VP级别的候选人。
你对这个领域的薪资行情、岗位需求、人才画像了如指掌。"
```

**核心分析维度：**
- 用户当前岗位在市场上的供需情况
- 同岗位"市场上最受欢迎的人"具备什么特质
- 用户如果再做1年，市场价值会怎么变
- 市场上对这个岗位的新趋势/新要求

---

### Agent D：头部企业专家 Agent（动态）

**做什么：** 从头部企业同岗位高级专家视角，分析用户与头部企业同级别的差距。

**动态生成逻辑：**
```
输入：用户的岗位 + 行业 + 意向公司
生成：一个头部企业同岗位的高级专家

示例（用户是B2B销售，意向公司是Salesforce）：
"你是Salesforce的高级销售专家，负责过企业级大客户销售。
你擅长：解决方案销售、客户关系管理、销售漏斗管理、跨部门协作。
你对B2B销售岗的能力模型、晋升标准、面试要求非常了解。"
```

**核心分析维度：**
- 头部企业同岗位的能力模型 vs 用户的差距
- 头部企业的工作方法论/思维框架，哪些可以迁移
- 用户如果要跳到头部企业同岗位，需要补什么
- 头部企业同岗位的日常工作节奏和思维方式

---

### Agent E：圆桌主持 Agent（固定）

**做什么：** 综合四个角色的观点，提炼共识和分歧，输出最终结论。

### Agent F：AI效能专家 Agent（固定）

**做什么：** 评估AI对该岗位的影响，给出AI学习路径建议。

**System Prompt 核心：**
```
你是一位AI效能专家，专注于评估AI对各岗位职业发展的影响。

分析维度：
1. 该岗位正在被AI如何改变？哪些能力会被替代？
2. 该岗位如何利用AI提效？哪些工作流可以用AI重构？
3. 该岗位的AI技能缺口在哪？需要掌握哪些AI工具/方法？
4. AI时代该岗位的竞争力模型有什么变化？
5. 未来2-3年，该岗位+AI会演变成什么形态？

输入：{career_portrait} + {current_work_info}
```

## 圆桌讨论设计

### 角色矩阵

| 角色 | 类型 | 视角 | 核心问题 |
|------|------|------|---------|
| 职业导师 | 固定(10%) | 发展视角+内部视角 | "你的天花板在哪？突破路径是什么？你在团队里是什么位置？" |
| 猎头 | 动态(50%) | 市场视角 | "你的市场价值多少？市场上什么样的人最受欢迎？" |
| 头部企业专家 | 100%动态 | 对标视角 | "你和头部企业同级别差距在哪？有哪些可以学？" |
| AI效能专家 | 固定(60%) | AI视角 | "你的岗位被AI替代的风险？如何用AI提效？" |
| 圆桌主持 | 固定(0%) | 综合 | 综合各方观点，提炼共识和分歧 |

### 讨论流程

```
1. 每个角色独立阅读用户的画像 + 当前工作信息
2. 每个角色给出自己的分析（200-300字）
3. 角色之间可以互相质疑和补充：
   - 管理者说"你在团队里是骨干" → 猎头说"但市场上你这个水平的人很多"
   - 头部企业专家说"你缺XX能力" → 规划师说"但这个能力在你当前公司不重要"
4. 主持Agent综合各方观点，提炼：
   - 共识：所有人都同意的提升方向
   - 分歧：不同角色看法不一致的地方
   - 最终建议：综合判断后的Top 3提升方向
```

## 这个模块的商业价值

| 维度 | 分析 |
|------|------|
| **覆盖面扩大** | 原来只服务"该跳"的用户（约30%），现在也服务"不该跳"的用户（约70%） |
| **复购可能** | 用户用了工作辅导 → 3个月后回来更新画像 → 看看情况有没有变 → 可能重新评估 |
| **口碑传播** | "这个AI不仅告诉我该不该跳，还告诉我怎么在当前岗位做得更好" — 口碑更好 |
| **圆桌差异化** | 4个角色（职业导师+猎头+头部企业专家+AI效能专家）的组合，市场上没有竞品这样做 |
