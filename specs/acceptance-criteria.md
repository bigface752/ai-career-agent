# PRD 补充：用户故事验收标准

> **文件职责**：验收标准。所有用户故事的Given/When/Then都记录在这里。
> 对应 PRD.md 第7节，为每个用户故事补充 Given/When/Then 验收标准

---

## 模块一：职业认知

### US-01 上传简历

**As a** 用户，**I want** 上传简历，**so that** AI了解我的背景。

**验收标准：**
```
Given 用户在上传页面
When 选择 PDF 文件（≤10MB）并点击上传
Then 系统开始解析，显示"正在分析你的简历..."

Given 用户上传了一个非 PDF 格式的文件
When 点击上传
Then 提示"请上传 PDF 格式的文件"

Given 用户上传了超过 10MB 的文件
When 点击上传
Then 提示"文件大小不能超过 10MB"
```

### US-02 确认解析信息

**As a** 用户，**I want** 确认AI解析的信息，**so that** 确保后续分析准确。

**验收标准：**
```
Given AI完成简历解析
When 展示解析结果（职位、行业、年限、技能、项目）
Then 用户可以看到所有解析字段，每个字段可编辑

Given 用户修正了某个字段
When 点击确认
Then 更新后的信息保存到画像系统
```

### US-03 快速诊断

**As a** 用户，**I want** 看到快速诊断结果，**so that** 知道这个产品有用。

**验收标准：**
```
Given 用户确认了解析信息
When 等待诊断完成（5 Agent × 3次并行，流式输出）
Then 显示：竞争力定性评级（强/中/弱）+ 一句话定位 + 薪资分位 + Top 2 短板 + Top 2 优势 + 各Agent评语

Given 快速诊断完成
When 用户看到结果
Then 结果可截图分享（卡片格式，含产品水印）
```

### US-04 引导式对话

**As a** 用户，**I want** 通过对话补充职业动机和价值观，**so that** 获得个性化建议。

**验收标准：**
```
Given 用户完成快速诊断
When 进入对话页面
Then AI基于简历信息提出第一个定制化问题

Given 用户回答了一个问题
When AI判断信息不充分
Then AI追问（最多追问3次）

Given 用户回答"不知道"或"随便"
When AI识别为无效回答
Then AI换一种方式重新提问，不写入画像

Given 对话进行到第4轮
When 用户回答完
Then 显示"初步发现"卡片（中间反馈）

Given 对话完成（4-6轮）
When 最后一轮结束
Then 生成职业画像，显示画像完成进度
```

### US-05 圆桌讨论

**As a** 用户，**I want** 看到圆桌讨论，**so that** 获得多视角分析。

**验收标准：**
```
Given 职业画像生成完成
When 触发圆桌讨论
Then 6个角色（心理学家+行业总监+职业导师+AI效能专家+猎头+面试辅导）并发生成发言

Given 圆桌讨论完成
When 展示结果
Then 顶部显示共识结论 + 分歧点，下方显示完整辩论过程

Given 用户想快速查看结论
When 点击"收起辩论，只看结论"
Then 只显示共识和分歧，隐藏辩论过程

Given 模块一圆桌讨论完成
When 触发结论判断
Then 3个模型独立分析，互相评议，最终生成"跳/观望/不跳"结论+个性化理由
```

### US-06 暂停对话

**As a** 用户，**I want** 暂停对话并稍后继续，**so that** 不被时间限制。

**验收标准：**
```
Given 用户在对话中途（第3轮）
When 关闭页面或退出
Then 对话进度保存，已回答的内容不丢失

Given 用户下次进入
When 系统检测到未完成的对话
Then 显示"你上次聊到第3轮，要继续吗？"
```

### US-07 画像完成进度

**As a** 用户，**I want** 看到画像完成进度，**so that** 知道还缺什么。

**验收标准：**
```
Given 用户完成模块一对话
When 查看画像页面
Then 显示"画像完成度 65%" + 已完成/未完成的维度列表

Given 用户使用了模块二
When 查看画像页面
Then 完成度更新为 80%
```

## 模块零：当前工作辅导

### US-08 明确评估结论

**As a** 用户，**I want** 看到明确的评估结论，**so that** 做出决策。

**验收标准：**
```
Given 模块一圆桌讨论完成
When 系统生成结论
Then 显示三个选项之一："建议跳槽" / "建议观望" / "建议留下"
   + 每个选项附带理由（2-3句话）

Given 结论为"建议留下"
When 显示结论
Then 同时显示"留任收益分析" + 提升方向概要
```

### US-09 选择是否接受结论

**As a** 用户，**I want** 选择是否接受结论，**so that** 保有决策权。

**验收标准：**
```
Given 系统显示结论为"建议留下"
When 用户看到结论
Then 显示两个按钮："接受，进入工作辅导" / "不接受，我要看机会"

Given 用户点击"不接受，我要看机会"
When 系统收到选择
Then 直接进入模块二（岗位匹配），不阻拦
```

### US-10 补充当前工作信息

**As a** 用户，**I want** 补充当前工作信息，**so that** 获得针对性建议。

**验收标准：**
```
Given 用户进入模块零
When 系统读取已有画像
Then 只追问 portrait_json 中缺失的字段（不重复问已知信息）

Given 对话完成
When 更新画像
Then portrait_json.current_work 字段填充完成
```

### US-11 模块零圆桌讨论

**As a** 用户，**I want** 看到4角色圆桌讨论，**so that** 从多视角分析当前工作。

**验收标准：**
```
Given 用户完成当前工作信息补充
When 触发圆桌讨论
Then 4个角色依次发言：职业导师 + 猎头 + 头部企业专家 + AI效能专家

Given 圆桌完成
When 展示结果
Then 显示共识结论 + 分歧点 + 每个角色的核心观点
```

### US-12 提升方案

**As a** 用户，**I want** 获得具体的提升方向和行动计划，**so that** 知道下一步做什么。

**验收标准：**
```
Given 圆桌讨论完成
When 生成提升方案
Then 显示：当前定位 + 天花板分析 + Top 3 提升方向（每个含具体行动+时间线）
   + 3/6/12月行动计划
```

## 模块二：岗位匹配

### US-13 输入JD

**As a** 用户，**I want** 通过粘贴文字或输入岗位名来输入JD，**so that** 获得匹配分析。

**验收标准：**
```
Given 用户在JD输入页面
When 粘贴完整的JD文字
Then AI解析JD，提取岗位要求

Given 用户只输入"某公司 销售总监"
When 点击分析
Then AI基于行业常识推断该岗位的典型要求，并提示"以下为推断结果，可能与实际JD有差异"

Given 用户输入的内容无法识别为有效JD
When 点击分析
Then 提示"无法识别岗位信息，请尝试粘贴完整的JD"
```

### US-14 匹配评分

**As a** 用户，**I want** 看到多维度匹配评分，**so that** 知道差距在哪。

**验收标准：**
```
Given JD解析完成
When 生成匹配报告
Then 显示：总体匹配定性评级（强/中/弱）+ 4个维度评级（技能/经验/薪资/发展）
   + 差距清单（含严重程度：大/中/小）
   + 优势清单
```

### US-15 简历优化建议

**As a** 用户，**I want** 获得简历优化建议，**so that** 提高匹配度。

**验收标准：**
```
Given 匹配报告生成
When 展示优化建议
Then 每条建议包含：改什么、怎么改、为什么改
   + 优先级排序（先改差距最大的部分）
```

### US-16 JD截图输入（V2）

**As a** 用户，**I want** 上传JD截图来输入岗位信息，**so that** 更方便。

**验收标准：**
```
Given 用户上传JD截图
When 系统处理
Then 使用 MiMo Vision 提取JD内容，展示提取结果供用户确认
```

## 模块三：面试辅导

### US-17 定制面试题

**As a** 用户，**I want** 获得基于JD定制的面试题，**so that** 有针对性地准备。

**验收标准：**
```
Given 用户选择目标岗位和面试轮次
When 生成面试题
Then 显示：专业题（3-5题）+ 行为题（2-3题）+ 非标准题（1-2题）
   每题标注类型和考察重点
```

### US-18 模拟面试追问

**As a** 用户，**I want** 像真实面试一样被追问，**so that** 练习应对追问。

**验收标准：**
```
Given 用户提交了面试回答
When AI评估回答质量
Then 基于回答内容追问（最多3次追问/题）
   追问类型：细节追问 / 数据追问 / 逻辑追问

Given 用户连续3轮给出敷衍回答
When AI判断回答质量低
Then 提示"你的回答比较简短，面试官可能会追问更多细节，试试展开说说？"

Given 用户中途关闭页面或1小时无操作
When 下次进入模块三
Then 显示"你上次答到第X题（共Y题），要继续吗？"
   选择"继续" → 从上次停下的题目继续
   选择"重新开始" → 清空进度，从第一题开始

Given 用户点击"放弃面试"
When 确认放弃
Then 已答题目保留，显示"继续面试"和"查看部分评估"选项

Given 用户放弃时已答题目 < 3题
When 选择"查看部分评估"
Then 提示"至少需要答完3题才能生成评估，建议继续"

Given 用户放弃时已答题目 ≥ 3题
When 选择"查看部分评估"
Then 进入答案评估，标注"部分完成（X/Y题）"，评级只基于已答题目

Given 面试session超过5天无操作
When 系统检测到
Then 提取已答内容到画像，删除session
```

> 详细状态机定义见 SPEC.md §3.15

### US-19 每题评分和优化建议

**As a** 用户，**I want** 看到每题的评分和优化建议，**so that** 知道怎么改进。

**验收标准：**
```
Given 模拟面试完成
When 展示评估结果
Then 每题显示：评分（1-5）+ 原始回答的问题 + 优化建议
```

### US-20 优化后回答示范

**As a** 用户，**I want** 看到优化后的回答示范，**so that** 知道怎么改。

**验收标准：**
```
Given 评估结果展示
When 用户查看某题详情
Then 显示优化后的回答示范（保持用户原始风格，不变成模板）
   + 标注使用了STAR法则的哪个部分
```

## 用户认证系统

### US-27 注册

**As a** 新用户，**I want** 用邮箱注册账号，**so that** 保存我的职业画像和对话记录。

**验收标准：**
```
Given 用户在注册页面
When 输入邮箱+密码+确认密码，点击注册
Then 系统发送6位验证码到邮箱

Given 用户输入正确的验证码
When 点击验证
Then 注册成功，自动登录，跳转到首页

Given 用户输入已注册的邮箱
When 点击注册
Then 提示"该邮箱已注册，请登录"

Given 用户输入的密码不符合规则
When 点击注册
Then 提示"密码至少8位，包含字母和数字"
```

### US-28 登录

**As a** 已注册用户，**I want** 用邮箱登录，**so that** 访问我的职业画像和对话记录。

**验收标准：**
```
Given 用户在登录页面
When 输入邮箱+密码，点击登录
Then 登录成功，跳转到首页

Given 用户输入错误的密码
When 点击登录
Then 提示"密码错误"（最多5次锁定30分钟）

Given 用户在其他设备登录
When 登录成功
Then 旧设备被踢掉，提示"账号在其他设备登录"
```

### US-29 忘记密码

**As a** 用户，**I want** 重置密码，**so that** 找回账号。

**验收标准：**
```
Given 用户在登录页面
When 点击"忘记密码"，输入邮箱
Then 系统发送6位验证码到邮箱

Given 用户输入正确的验证码+新密码
When 点击重置
Then 密码重置成功，跳转到登录页面
```

## 对话状态管理

### US-30 暂停对话

**As a** 用户，**I want** 暂停对话并稍后继续，**so that** 不被时间限制。

**验收标准：**
```
Given 用户在对话中途（第3轮）
When 关闭页面或退出
Then 对话进度保存，已回答的内容不丢失

Given 用户1小时无操作
When 系统检测到
Then 自动暂停对话，显示"你的对话已暂停"
```

### US-31 恢复对话

**As a** 用户，**I want** 恢复之前的对话，**so that** 继续未完成的对话。

**验收标准：**
```
Given 用户登录后进入对话页面
When 系统检测到有未完成的对话
Then 显示"你上次聊到第X轮，要继续吗？"

Given 用户点击"继续"
When 系统加载对话历史
Then 恢复对话，显示历史消息，继续当前轮次
```

### US-32 超期清理

**As a** 系统，**I want** 清理超期对话，**so that** 释放存储空间。

**验收标准：**
```
Given 对话暂停超过5天
When 系统检测到
Then 发送站内+邮件通知用户"你的对话即将过期"

Given 用户48小时内未响应
When 系统执行清理
Then 提取关键信息到画像，删除对话

Given 用户48小时内回来
When 用户点击"继续"
Then 恢复对话，不清理
```

## neat-freak机制

### US-33 对话结束提炼

**As a** 系统，**I want** 对话结束后自动提炼要点，**so that** 保持画像时效性。

**验收标准：**
```
Given 对话结束
When 系统执行提炼
Then 识别新信息、变化信息、矛盾信息

Given 提炼完成
When 用户查看
Then 显示"画像已更新"通知，用户可查看新增/更新的字段
```

### US-34 15轮提醒

**As a** 系统，**I want** 对话进行到15轮时提醒用户，**so that** 用户决定是否更新画像。

**验收标准：**
```
Given 对话进行到第15轮
When 用户回答完
Then 显示"我们已经聊了15轮了，要不要更新一下你的职业画像？"

Given 用户点击"更新"
When 系统执行提炼
Then 画像更新，对话继续

Given 用户点击"继续"
When 用户继续对话
Then 对话继续，不提炼
```

## 前端状态管理

### US-35 流式输出

**As a** 用户，**I want** 看到AI在"打字"，**so that** 有对话感。

**验收标准：**
```
Given 用户发送消息
When AI回复
Then 显示"正在输入..."，然后逐字显示回复（100ms/字）

Given 网络较慢
When AI回复
Then 降级为整块显示，不卡顿
```

### US-36 打字机效果

**As a** 用户，**I want** 看到圆桌讨论的打字机效果，**so that** 有"看直播"的感觉。

**验收标准：**
```
Given 圆桌讨论开始
When 各角色发言
Then 每个角色的发言逐字显示（100ms/字）

Given 某个角色发言完成
When 下一个角色开始发言
Then 显示角色切换动画
```
